## P0 â€” Critical (bugs causing silent failures)

### 1. `main()` never ran via backward-compat symlink in `sync_dotfiles.sh` *(self-introduced regression, now fixed)*

- **File:** `scripts/sync/sync_dotfiles.sh:821`
- Previous fix changed `SELF_PATH` to use `readlink -f`, breaking the `[[ "$SELF_PATH" == "$ZERO_PATH" ]]` guard. `main()` was silently skipped whenever the script was invoked via `scripts/sync_dotfiles.sh` (the symlink) instead of the real path.
- **Fix applied:** Changed guard to standard `[[ "${BASH_SOURCE[0]}" == "${0}" ]]` idiom. Verified `main` now runs via both paths.

### 2. `install_fonts.sh` computes wrong `ENV_DIR` when invoked via real path
- **File:** `scripts/install/install_fonts.sh:4-5`
- `ENV_DIR="$(dirname "$SCRIPT_DIR")"` only goes up 1 level from `scripts/install/`, yielding `/home/yashar/env/scripts`. Then `source "$ENV_DIR/functions/common_funcs"` fails (no such file). Caught by `|| warning` in `setup.sh:1003`, so **fonts silently fail to install**.
- **Fix:** Same pattern as the sync scripts:
  ```bash
  SCRIPT_DIR="$(cd "$(dirname "$(readlink -f "${BASH_SOURCE[0]:-$0}")")" && pwd)"
  ENV_DIR="$(cd "$(dirname "$SCRIPT_DIR")/.." && pwd)"
  ```

### 3. `setup.sh` can infinite-loop when invoked via real path or curl-pipe
- **File:** `scripts/install/setup.sh:42-43,86-87`
- "Are we in repo?" check uses `$SCRIPT_DIR/../config/repo.conf` â€” only one `../` from `scripts/install/` (needs `../../`). After curl-pipe clones & re-execs `bash "$ENV_TARGET/scripts/install/setup.sh"`, the check fails again â†’ `NEEDS_CLONE=1` â†’ `.git` exists so skips clone â†’ re-execs â†’ infinite loop.
- **Fix:** Change `../` to `../../` on lines 42-43; fix `ENV_DIR="$(dirname "$SCRIPT_DIR")"` on line 87 to go up 2 levels.

### 4. `setup.sh` writes broken crontab referencing non-existent `scripts/backup.sh`
- **File:** `scripts/install/setup.sh:888`
- Every fresh install produces a cron entry for a script that doesn't exist.
- **Fix:** Either implement `scripts/sync/backup.sh` or remove the line.

### 5. `sync_encrypted.sh` double `trap â€¦ EXIT` leaks age identity file containing `AGE_SECRET`
- **File:** `scripts/sync/sync_encrypted.sh:30` + `:1248`
- Line 30 sets `trap _cleanup_age_identity EXIT`; line 1248 sets `trap cleanup EXIT`, which **replaces** the first trap (bash only allows one handler per signal). `cleanup()` only removes `$TEMP_DIR`, not the `mktemp`-created identity file. Secret key persists in `/tmp` until reboot.
- **Fix:** Chain them: `trap '_cleanup_age_identity; cleanup' EXIT`, or call `_cleanup_age_identity` from inside `cleanup`.

### 6. `bw_funcs` `add_secret` / `get_secret` are incompatible
- **File:** `functions/bw_funcs:13-28` vs `:41-44`
- `add_secret` creates a **secure note** (type 2) with the secret in `.notes`. `get_secret` reads `.login.password` (null for secure notes). Stored secrets can never be retrieved.
- **Fix:** Make `add_secret` create a login item with `.login.password`, OR make `get_secret` read `.notes` for type-2 items.

### 7. `file_chat_funcs` typo breaks the env-override mechanism
- **File:** `functions/file_chat_funcs:2`
- `PRIVATE_CHAT_FILE=${CHPRIVATE_CHAT_FILEAT_FILE:-...}` â€” looks like a merge artifact (mixed `PRIVATE_CHAT_FILE` + `CHAT_FILE`). Variable can never be set, so the override is dead.
- **Fix:** `PRIVATE_CHAT_FILE=${PRIVATE_CHAT_FILE:-"$HOME/.private_chatfile"}`.

## P0 â€” Critical (security)

### 8. Hardcoded camera credentials in `cam_ctrl.py`
- **File:** `scripts/tools/cam_ctrl.py:25-28`
- `CAM_IP="192.168.1.140"`, `CAM_USER="admin"`, `CAM_PASS="1111"`. Same creds at `:294` (ONVIF digest). Password is in git history forever.
- **Fix:** Read from env vars (`CAM_USER`/`CAM_PASS`) like `show_cam.sh` already does. **Rotate the password.**

### 9. `cam_web.py` exposes unauthenticated camera PTZ controls on `0.0.0.0`
- **File:** `scripts/tools/cam_web.py:44,179,186,206`
- Binds `0.0.0.0` with no auth + `Access-Control-Allow-Origin: *`. Anyone on the LAN can pan/tilt/zoom the camera, read presets, or capture snapshots.
- **Fix:** Bind `127.0.0.1` by default; add a token check; drop wildcard CORS.

### 10. `letmein.sh` uses TLS `verify=0` while spawning a remote shell
- **File:** `scripts/tools/letmein.sh:20,30,43,54`
- All `socat OPENSSL` connections disable cert verification with an ephemeral self-signed cert â†’ MITM-able by anyone on the network. Acceptable for trusted LAN only, but should be documented loudly or pinned.
- **Fix:** Pin the cert (pass `cafile=`) or document the trust model.

### 11. JSON injection via unescaped interpolation
- **`functions/pingme:13-14`** â€” Telegram payload: `"{\"text\": \"$message\"}"` breaks if `$message` contains `"`, `\`, or newlines.
- **`scripts/tools/generate_image.sh:100-104`** â€” same pattern for image-gen API.
- **Fix:** Use `jq -nc --arg ... '{...}'`.

### 12. Secrets visible in process arguments
- **Files:** `functions/monitors:31,96`, `functions/common_funcs:103`, `scripts/sync/sync_encrypted.sh:729-730`
- Tokens passed as `curl -u ":$TOKEN"` or `-H "Authorization: bearer $TOKEN"` are visible via `ps`.
- **Fix:** Use `curl --config -` with creds via stdin, or `curl -H @<file>`.
